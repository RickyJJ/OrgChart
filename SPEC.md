# 📖 《青云志》项目规范 (SPEC)

## 1. 业务目标与验收标准

### 1.1 业务目标
- **产品名称**: 《青云志》 (The Path to Court)
- **核心目标**: 打造一个沉浸式的中国古代官制文化探索平台，科普古代官职的层级结构、实际职能、代表人物、历史典故与经典诗词。
- **目标用户**: 传统文化爱好者、历史系/文科学生、对古代职场及诗词典故感兴趣的大众用户。
- **MVP阶段目标**: 以**唐代**官职体系为切入点，完成核心的“官阶架构展示”、“诗词典故关联”以及基本的UI风格搭建。

### 1.2 验收体验标准 (UI/UX)
- **整体架构**: 左侧深色导航侧边栏 (Sidebar Navigation)，右侧为带传统画框的宣纸主内容区 (Framed Content Area)。
- **整体风格**: 新中式极简风 (Neo-Chinese Minimalist) 与“水墨卷轴”结合。
- **色彩规范**: 
  - 侧边栏及主字色：藏青/玄黑 (#1e293b, #1a1a1a)。
  - 主背景色：宣纸暖白底色附带暗纹 (#f5f5dc 变体)。
  - 强调色：朱砂红 (#af292e，用于印章、核心按钮、活动状态标签)。
- **排版与字体**: 标题和关键层级使用优雅的宋体 (Serif)，正文使用易读的黑体 (Sans-serif)。支持双语显示（中文主标题，英文副标题）。
- **细部控件特征**:
  - **根节点 (皇权)**: 使用高保真背景图 `emperor-bg.png`，且不显示标题文字和边框。
  - **朝代切换按钮**: 采用行楷字体排版。选中态背景为极具包裹感的专属印泥图片 (`switch-bg-1.png`)，点击时伴随专属“盖印 (Stamp)”原地位移缩放动效，文字始终维持深墨色。
  - **官职卡片**: 极简牌匾风格（尺寸由比例计算设定为 `60px * 167px`），强制使用纯中文竖版排版，字体设定为 `font-serif` (Noto Serif SC 1.6rem, Weight 600)，标题过长（超过 4 个字）时显示秒响应的悬停走马灯（Marquee）特效（单次滚动并平滑还原，速度根据字数自适应，保持匀速感）。
  - **典故呼吸印记 (Lore Breathing Mark)**: 当接口数据表明该官职含有丰富的“典故连珠”数据（如关联著名历史人物或经典诗词）时，自动在卡片边缘点缀一枚极微小的朱砂红 (#af292e)  动态印章或水墨晕染点。该印记需自带极缓的呼吸动效（Opacity 0.6 ↔ 1.0 循环），作为“此处有故事”的隐性视觉引导（Affordance），激发用户探索欲。
  - **三明治式渲染层级**: 采用多层叠加结构——底层为 `card-bg-insight.png` 水墨底纹（带 Alpha 遮罩裁剪），中层为 `card-border-overlay` 墨灰色框线，顶层为文字层，通过 `background-size: 100% 100%` 实现零形变渲染。
  - **扇形展开动效 (Fan-out)**: 子节点收起时，父节点背后左右各以 `3deg` 角度旋转展开两张伪卡片，取代旧的横向堆叠方案，视觉更具古风韵律感。
  - **悬浮详情卡片**: 改为在右侧悬浮的立体卡片，包含全量英文翻译、引言关联列表等，消除主架构图的中英混排冗余。
- **核心动效**:
  - 朝代切换时的“双向缩放过渡”反馈。
  - 长名称官职悬停时的自动纵向走马灯（仅滚动一次）。
  - 初次加载时的“画卷平铺”入场动画 (Implemented).

## 2. 功能详细规则

### 2.1 朝代切换与全局视界 (Dynasty Navigator)
- 支持从顶部下拉菜单或横向时间轴快速切换不同的朝代（如唐代、明代、宋代等）。
- 切换朝代时，平滑加载该朝代的中央/地方官制结构树。

### 2.2 官阶巡礼 (The Hierarchy Tree)
- **功能描述**: 多维组织架构图，直观展示三省六部等古代国家机器的运转模式。
- **交互与技术实现**:
  - 基于原生前端技术 (Vanilla JS + CSS) 渲染交互式树状图，以完美契合水墨美术风格。
  - 支持横向/纵向拖拽浏览。
  - 支持节点点击展开与折叠，以便在庞杂的官制中保持视线清晰。
- **MVP 范围**: 优先实现并完善“唐代”与“明代”中央官制的树状图（如尚书省、中枢省、内阁及下辖机构）。

### 2.3 双层信息架构与交互 (Two-Level Detail UX)
- **第一层 (内联卡片)**: 
  - 样式：白底水墨晕染卡片，包含中文名及英文翻译（如：中书省 Secretariat）。
  - 动作：点击选中卡片，触发边缘发光高亮，并呼出悬浮详情卡片。**支持重复点击切换（Toggle）开启/关闭状态。**
- **第二层 (悬浮详情卡片)**:
  - 触发：点击第一层卡片。
  - 内容：在屏幕右侧弹出的悬浮立体面板，深度展示该官职的维度数据。**点击面板以外的空白区域或再次点击原卡片可将其关闭。**

### 2.4 四维文化数据展示 (Four-Dimensional Cultural Lore)
根据样图 `detail-panel-sample.png` 指引，悬浮详情卡采用流式垂直布局：
1. **内容显示 (Header)**: 
   - 展示大字号宋体“官职名称”，紧邻右侧展示中字号宋体“品阶”。
   - 品阶下方展示灰度英文翻译 (Italicized Serif)。
2. **职能详情 (Summary)**: 
   - 样式：深灰色黑体，行高 1.6。内容展示职责原文。
3. **官职专属插图 (Illustration)**:
   - 视觉：低饱和度水墨化，边缘柔和羽化渐隐，完美融入宣纸背景。
4. **典故连珠 (Lore Pearls)**: 
   - 布局：左侧放置“毛笔/卷轴”融合图标，右侧为模块标题“典故连珠”。
   - 展现：采用大字号、沉浸式宋体展示诗词/名言。右下角点缀一枚极小红印。
   - 展开：超过 3 行自动出现渐变遮罩与“...展开阅读”按钮。
5. **风流人物 (Figures)**: 
   - 排版：小字号、灰度。横向流式布局，名字间以顿号分隔。
6. **互动入口 (CTA)**: 
   - 底部横向撑满的朱砂红 (#af292e) 圆角按钮，文案“尝试此官职”，白字。

### 2.7 详情面板交互与边缘情况 (Detail Panel UX & Edge Cases)
- **动态显隐**: 若无“典故”或“人物”，相关模块隐藏，插图与 CTA 按钮平滑衔接。
- **防止穿透**: 阻止点击/滚动事件冒泡至底层架构树。
- **出场动画**: 300ms ease-out 右侧滑入 + 渐现。

### 2.5 全局检索与典故中心 (Global Search & Lore Center)
1. **功能定位**: 打破单一组织架构的枯燥感，作为贯穿全站的“文化虫洞”。实现“人-官-文”的深度联动，允许用户跨越层级直接穿透定位到目标官职。
2. **入口与形态**: 功能深度集成于全局检索面板中。用户通过点击侧边栏顶部的“搜索”图标触发检索面板，内容涵盖所有官职、人物及典故。原独立的“典故连珠”标签页已根据需求移除，以保持导航栏精简。
3. **多维模糊检索**:支持按 官职名（如“司马”）、历史人物（如“白居易”）、诗词典故（如“江州司马”）三个维度进行跨朝代搜索。**搜索逻辑需忽略关键词及被搜索内容中的空格（如输入“吏部”应能匹配“吏  部”）。**
4. **结果展示**:搜索结果以精简列表呈现，包含：官职名称、品阶、简短的职权白话翻译  以及匹配到的高亮命中词。
5. **穿透联动**:
  - 视界定位: 用户点击检索结果后，主界面的“官阶巡礼”树状图需自动平滑移动（Pan）并缩放（Zoom），将目标官职节点居中显示。
  - 状态联动: 目标节点自动触发边缘发光高亮选中态。
  - 详情呼出: 定位完成后，右侧自动弹出悬浮详情卡片 ，直接向用户展示该官职完整的四维文化数据 （含历史配图、人物及引言典故 ）。
### 2.6 “入仕”模拟器 (Modern-to-Ancient Job Matcher)
- **功能描述**: 趣味互动模块，打破古今壁垒。用户输入现代职业，系统匹配最贴合的古代官职，增强社交传播属性。
- **交互流程**:
  1. 提供极简输入框（如用户输入“程序员”、“产品经理”、“财务总监”）。
  2. 系统通过预设的映射字典（Mapping Dictionary），智能输出对应的古代官职（如：“程序员” -> “将作监丞”，“产品经理” -> “中书舍人”或“考功司员外郎”）。
  3. **视觉反馈**: 动态生成一张带有新中式边框、**朱红印章**盖印动效的“委任状”海报。
  4. 支持前端使用 HTML-to-Image (如 `html2canvas`) 技术实现海报保存，支持用户长按保存或分享。

## 3. 架构与数据模型

### 3.1 核心技术栈与架构
- **Frontend**: React 18, Vite.
- **Styling**: TailwindCSS (Custom configuration for Neo-Chinese theme variables).
- **Icons**: Font Awesome 6 (React components) or SVG.
- **Fonts**: Noto Serif SC, Inter (Google Fonts).
- **Architecture Highlights**: 
  - **视觉-逻辑解耦 (Visual-Logical Decoupling)**: 引入稳定的 `CardContainer` 容器层。树状连线的坐标计算仅依赖该静态容器，而视觉动效（如 Hover 上浮、选中缩放）仅在容器内部发生。这确保了交互过程中连线的绝对稳定性。
  - **递归可见性检查 (Recursive Visibility)**: 树状连线渲染器会递归检查祖先节点的展开状态，确保折叠子树内部的连线被物理隐藏，避免“虚影线”残留。
- **Structure**: Component-based architecture. State managed via React hooks. `data.js` will be converted to a module or JSON.
- **Tools**: `html2canvas` (or similar) for the poster generation module.

### 3.2 数据结构规范 (Data Model)
所有的朝代数据需遵循严格的 JSON schema `node`：

```typescript
interface Node {
    id: string;
    title: string;        // 官职名称 (如: 中书省)
    type?: string;        // 官职类别标记 (如: admin, military, finance) => 映射小图标
    level: string;        // 官阶 (如: 正二品)
    description: string;  // 详细/摘要职责
    figures?: string[];   // 代表人物数组
    allusions?: Array<{ title: string, text: string }>;  // 典故数组
    poetry?: Array<{ author: string, poem: string }>;    // 诗词数组
    children?: Node[];    // 下属机构/官职
}
```

## 4. 图像资产规范 (Image Asset Convention) - 【AGENT 执行准则】

### 4.1 详情面板资产要求
- **宣纸底纹 (Paper Texture)**: 
  - **路径**: `/public/assets/ui/paper-texture.png` (或复用 `main-bg.png`)。
  - **渲染**: 必须平铺且无缝，确保在不同高度的卡片下背景质感一致。
- **水墨羽化遮罩 (Ink Mask)**:
  - **路径**: `/public/assets/ui/ink-mask-large.png`。
  - **效果**: 必须是 8-bit Alpha 通道的 PNG，边缘呈现不规则的水墨飞白，用于卡片边缘和插图边缘的渐隐。
- **官职插图 (Illustrations)**:
  - **路径**: `/public/assets/content/[role_name].png`。
  - **风格**: 必须保持**低饱和度、水墨渲染**感。
  - **处理**: 严禁拉伸，强制使用 `object-fit: contain`。必须配合 `-webkit-mask-image` 实现边缘羽化，禁止出现直角边框。

### 4.2 存储与读取规则
- **严禁硬编码路径**: 详情面板的背景图、遮罩图需引用 CSS 变量或全局常量。
- **动态读取**: 业务插图（Figure/Illustration）必须从 `data.js` 的 `panelImage` 字段读取，严禁在组件内直接 `import` 业务图片。
- **缺省处理**: 若字段为空，必须回退至 `/public/assets/content/horse_rider.png`。
